# Contributing

## Naming convention

Commit messages and pull request titles must be written in English and follow
this format:

```text
type(scope): description
```

Allowed types:

- `feat`: new functionality.
- `fix`: bug fix.
- `docs`: documentation.
- `style`: visual or formatting changes.
- `refactor`: internal changes that preserve behavior.
- `perf`: performance improvement.
- `test`: tests.
- `build`: build system or dependency changes.
- `ci`: continuous integration.
- `chore`: maintenance.
- `revert`: revert a previous change.

Examples:

```text
feat(navbar): implement responsive navigation
fix(navbar): replace hamburger icon with SVG
docs(project): document contribution naming convention
```

Use imperative descriptions that start in lowercase and have no trailing
period. Uppercase acronyms such as `SVG` are allowed.
